# Zgine Engine Vision

版本日期：2026-05-25

## 定位

Zgine 是用于学习游戏引擎底层细节的 C++20 自研 3D 游戏引擎。它不是 Unity/Unreal 的生产级替代品，而是一个工程化的教学引擎：每个模块都应尽量展示真实引擎会遇到的问题，同时保持代码规模和复杂度可控。

## 当前目标

短期目标是把已有 MVP 稳定成“可制作小型游戏的最小完备引擎”：

- Runtime 和 Editor 边界清晰。
- ECS、场景、序列化、资源、渲染、物理、音频、脚本有统一生命周期。
- Editor 能创建、保存、加载、检查和运行场景。
- OpenGL 保持 reference backend。
- Vulkan 作为跨平台图形 API 教学主线逐步实现。

## 非目标

当前阶段不追求：

- 完整 AAA 渲染器。
- 生产级多平台发布管线。
- 大型团队权限/协作系统。
- 复杂渲染图、动画图、蓝图系统的一次性完整实现。

## 设计取舍

- 优先显式边界，而不是过度抽象。
- 优先可验证小步，而不是一次性大重构。
- 优先教学清晰度，而不是所有性能优化。
- 允许阶段性 stub，但 stub 必须明确、可测试、可替换。

## 开发方法

Zgine 使用：

- OpenSpec：管理每一次功能变更。
- Constitution：约束项目底线。
- BMAD：只用于 Renderer、Asset Pipeline、Prefab、Editor Play Mode 等复杂大模块的设计阶段。

这套流程的目标是减少长期 AI 编码和人工重构中的上下文漂移。
